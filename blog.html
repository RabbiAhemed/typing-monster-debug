Q1. Local Storage vs Session Storage?

Local storage and session storage are almost similar but there is a bit difference.Local storage and session storage are used to store data in the browser Session storage’s data is wiped or cleared when a particular session or tab is closed.and opening a new tab creates a new session storage for that particular tab.if the tab or browser is closed the session storage will be lost.Session storage is for one browsing session and it is removed as soon as the user closes the tab where that session was set. While local storage is around forever until the user deletes it.In a word session storage’s data is persisted until the tab or window is closed. The main difference is that local storage’s data does not expire,the data is shared across, data is persisted even after closing the session or tab.local storage’s objects properties can be set by using setItem() and then can be accessed/retrieved by using getItem().Also deletion/removal is possible using removeItem()

Q2. Global scope vs Block scope?

A variable declared at the top of a program or outside of a function is considered a global scope variable. The global variables can be accessed by any line of code in the program, including inside blocks.

The block scope can be defined as the scope of the variables inside the curly brackets {}. Now, these curly brackets can be of loops, or conditional statements, or something else. It is only allowed to refer to these variables from within the curly brackets {}.
